# 🐕🩺 Dr. Dog — 반려견 증상 자가진단 서비스

반려견 보호자가 강아지의 증상을 입력하면 **예상 질병과 즉각적인 대처법**을 안내해주는 모바일 반응형 웹앱입니다.

---

## 🌐 서비스 바로가기

> **GitHub Pages 배포 주소** (레포지토리 이름에 맞게 주소가 자동 생성됩니다)

---

## 📱 주요 기능

| 기능 | 설명 |
|---|---|
| **기본 정보 입력** | 품종 자동완성 검색, 나이 슬라이더, 몸무게 입력 → 자견/성견/노령견 자동 분류 |
| **SVG 신체 부위 선택** | 강아지 실루엣을 직접 탭하거나 탭 버튼으로 부위 선택 |
| **증상 카드 다중 선택** | 30개 이상의 증상 카드, 기타 직접 입력, 복수 선택 가능 |
| **심각도 & 기간 입력** | 경미함/보통/심각함, 오늘 처음/2~3일/일주일 이상 선택 |
| **질병 예측 알고리즘** | 나이·품종·체형·심각도 가중치 기반으로 최대 3가지 예상 질병 제시 |
| **긴급도 판정** | 🟢 안심 / 🟡 주의 / 🔴 긴급 3단계 자동 판정 및 배너 표시 |
| **대처법 안내** | DO / DON'T 아코디언 카드 ("왜요?" 버튼으로 이유 설명 확인) |
| **병원 제출 리포트** | 수의사 전달용 증상 요약 텍스트 자동 생성 + 원클릭 클립보드 복사 |
| **구글 지도 연동** | 📍 내 위치 기반 24시 동물병원 검색 (구글 지도 링크 연동) |

## 🚀 GitHub Pages 배포 방법 (처음 하시는 분)

### 1단계: GitHub 계정 만들기
[github.com](https://github.com)에서 무료 계정을 만드세요.

### 2단계: 새 레포지토리(저장소) 만들기
1. GitHub 홈에서 초록색 **[New]** 버튼을 클릭합니다.
2. **Repository name**에 원하는 이름을 입력합니다. (예: `dog-doctor`)
3. **Public**으로 설정하고 **[Create repository]** 버튼을 누릅니다.

### 3단계: 파일 업로드
1. 생성된 레포지토리 화면에서 **[uploading an existing file]** 링크를 클릭합니다.
2. `dog-doctor` 폴더 안의 **모든 파일과 폴더**(`.github` 폴더 포함)를 드래그해서 올립니다.
3. 아래 **[Commit changes]** 버튼을 눌러 업로드를 확정합니다.

### 4단계: GitHub Pages 활성화
1. 레포지토리 상단 탭에서 **[Settings]** 를 클릭합니다.
2. 왼쪽 메뉴에서 **[Pages]** 를 클릭합니다.
3. **Source** 항목을 **GitHub Actions** 로 변경합니다.
4. 이제 파일을 업로드할 때마다 자동으로 웹사이트가 배포됩니다!

### 5단계: 배포 주소 확인
- 설정 후 1~2분 기다리면 **Settings → Pages** 화면에 `https://[사용자이름].github.io/[레포지토리이름]/` 형태의 주소가 나타납니다.
- 해당 주소를 브라우저에서 열면 서비스를 확인하실 수 있습니다!
